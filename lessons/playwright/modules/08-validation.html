Module 8: Validation

Beginner Friendly This module explains concepts in plain language and then gives practical examples.

What You Will Learn

How to validate UI behavior in meaningful ways using Playwright assertions.

Simple Explanation

Validation means confirming expected outcomes with assertions. In Playwright, assertions are auto-retrying, so they wait for UI to reach expected state before failing.

Real-World Example

After creating an order, you should verify success message plus new row in order table with correct order id.

Step-by-Step Example (Code)

// UI assertions (auto-retrying)
      await expect(page).toHaveURL(/dashboard/);
      await expect(page.getByRole('heading', { name: 'Dashboard' })).toBeVisible();
      await expect(page.locator('.total')).toContainText('100');

      // Non-UI assertion (immediate check)
      const tax = 18;
      const amount = 100;
      await expect(amount + tax).toBe(118);

      // Soft assertion lets test continue and reports at end
      await expect.soft(page.getByTestId('optional-banner')).toContainText('Offer');

Common Assertion Patterns You Should Know

await expect(locator).toBeVisible();
      await expect(locator).toBeHidden();
      await expect(locator).toHaveText('Order placed');
      await expect(locator).toHaveCount(3);
      await expect(page).toHaveTitle(/Dashboard/);
      await expect(page).toHaveURL(/orders/);

      // Useful custom failure message
      await expect(page.getByRole('button', { name: 'Pay now' }), 'Pay button should be visible before final submit').toBeVisible();

Beginner Notes (Line by Line)

  1. await expect(page).toHaveURL(/dashboard/); - Validates navigation outcome by matching URL pattern.
  2. await expect(...).toBeVisible(); - Checks user-visible state, not just element existence in HTML.
  3. await expect(...).toContainText('100'); - Validates dynamic values shown in UI.
  4. await expect(amount + tax).toBe(118); - You can also validate plain TypeScript calculations.
  5. expect.soft(...) - Records failure but allows remaining test steps to run.

Common Mistake to Avoid

Checking only one generic condition (like title) and assuming feature works. Use multiple assertions: URL, visibility, business data, and state change.

Practice Exercise

  1. Add URL, visibility, and value assertions in one scenario.
  2. Validate a success toast and matching row data.
  3. Add one soft assertion and one custom assertion message.
