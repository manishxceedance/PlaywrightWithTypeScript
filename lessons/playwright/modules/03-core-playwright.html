Module 3: Core Playwright

Beginner Friendly This module explains concepts in plain language and then gives practical examples.

What You Will Learn

How tests are organized, how assertions validate outcomes, and how hooks/fixtures reduce repeated setup.

Simple Explanation

test() defines one scenario, describe() groups related scenarios, expect() checks results, and hooks (beforeEach/afterEach) prepare or clean test state.

Real-World Example

You can group all checkout tests in one describe block, so anyone can quickly understand that feature coverage.

Step-by-Step Example (Code)

import { test, expect } from '@playwright/test';

test.describe('Login', () => {
  test.beforeEach(async ({ page }) => {
    await page.goto('https://example.com/login');
  });

  test('valid login', async ({ page }) => {
    await expect(page).toHaveURL(/login/);
  });
});

Extra Essential Topic: Hooks and Fixtures

This is one of the most important ideas for real projects. Put repeated setup in hooks or fixtures so every test stays short and clean.

import { test, expect } from '@playwright/test';

test.describe('Cart', () => {
  test.beforeEach(async ({ page }) => {
    await page.goto('/');
    await page.getByRole('button', { name: 'Accept cookies' }).click();
  });

  test('add one item', async ({ page }) => {
    await page.getByRole('button', { name: 'Add to cart' }).first().click();
    await expect(page.getByTestId('cart-count')).toHaveText('1');
  });

  test('remove item', async ({ page }) => {
    await page.getByRole('button', { name: 'Add to cart' }).first().click();
    await page.getByRole('button', { name: 'Remove' }).click();
    await expect(page.getByTestId('cart-count')).toHaveText('0');
  });
});

  1. import { test, expect } from '@playwright/test'; - This line brings in Playwright's test tools so you can write and run tests.
  2. test.describe('Login', () => { - This groups related test cases for the Core playwright feature.
  3. test.beforeEach(async ({ page }) => { - This setup runs before each test to make sure the browser is ready.
  4. await page.goto('https://example.com/login'); - This opens the web page you want to test.
  5. }); - This is a supporting line for the Core playwright example.
  6. test('valid login', async ({ page }) => { - This line starts a new test scenario for Core playwright.
  7. await expect(page).toHaveURL(/login/); - This checks if what happened matches what you expected.
  8. }); - This is a supporting line for the Core playwright example.
  9. }); - This is a supporting line for the Core playwright example.

Common Mistake to Avoid

Writing long tests with many unrelated checks. Keep one test focused on one behavior.

Practice Exercise

  1. Create one describe block with two focused tests.
  2. Add one beforeEach hook and at least two expect validations.
