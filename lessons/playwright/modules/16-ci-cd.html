Module 16: CI/CD

Beginner Friendly This module explains concepts in plain language and then gives practical examples.

What You Will Learn

How to run automation in pipelines and publish artifacts.

Simple Explanation

CI runs tests automatically for pull requests and nightly builds. This gives quick feedback before code reaches production.

Real-World Example

Developer opens PR, smoke tests run in CI, failure blocks merge until issue is fixed.

Step-by-Step Example (Code)

npx playwright test --headed=false
Line-by-line explanation and CI setup guide:
  1. npx playwright test --headed=false — Runs tests in headless mode (no visible browser window). CI servers have no display, so headless is required. It also runs faster than headed mode.
  2. Typical GitHub Actions pipeline steps:
  3. Step 1: npm ci — Installs exact dependency versions from package-lock.json. Faster and more reliable than npm install in CI.
  4. Step 2: npx playwright install --with-deps — Downloads Playwright browser binaries AND their system-level dependencies (fonts, media codecs) on the CI runner. This step is required because CI machines do not have browsers pre-installed.
  5. Step 3: npx playwright test — Runs the full test suite. Exit code 0 means all passed; non-zero causes the pipeline to fail.
  6. Step 4: Upload playwright-report/ as a CI artifact — Allows team members to download and view the HTML report without needing to re-run the tests locally.
  7. Smart CI strategy: Run smoke tests on every pull request (fast, blocks bad merges). Run full regression on a nightly schedule (thorough, not blocking every PR).

Beginner Notes

  1. CI gives every code change an automatic quality gate — failing tests block the merge.
  2. Always upload test reports as CI artifacts so failures can be investigated without re-running.
  3. Smoke on PR + full regression on schedule is the industry-standard CI strategy for test automation.

Common Mistake to Avoid

Running full regression on every tiny change. Use smoke on PR and full regression on schedule.

Practice Exercise

  1. Create a CI job for smoke tests on pull request.
  2. Upload report and trace artifacts for failed tests.
