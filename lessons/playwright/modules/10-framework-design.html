Module 10: Framework Design

Beginner Friendly This module explains concepts in plain language and then gives practical examples.

What You Will Learn

How to design scalable and maintainable test framework structure.

Simple Explanation

POM stores page interaction logic in one place, so test files stay clean and readable.

Real-World Example

If login button text changes, update locator once in LoginPage class instead of many test files.

Step-by-Step Example (Code)

import { Page } from '@playwright/test';

export class LoginPage {
  constructor(private readonly page: Page) {}

  async login(email: string, password: string) {
    await this.page.getByLabel('Email').fill(email);
    await this.page.getByLabel('Password').fill(password);
    await this.page.getByRole('button', { name: 'Sign in' }).click();
  }
}
Line-by-line explanation:
  1. import { Page } from '@playwright/test'; — Imports the Page type from Playwright. This type annotation is required by TypeScript so the IDE knows what methods are available on this.page, enabling autocomplete and compile-time type checking. Without this import, TypeScript will report "Cannot find name 'Page'" and the file will not compile.
  2. export class LoginPage { — Defines a reusable Page Object class. export makes it importable in test files. All login-related locators and actions live here — tests never need to know the implementation details.
  3. constructor(private readonly page: Page) {} — TypeScript constructor that receives the Playwright page object from the test. Stored as this.page so all methods in the class can use it.
  4. async login(email: string, password: string) — A reusable method that encapsulates the full login sequence. Tests call this one line instead of repeating three separate actions every time.
  5. await this.page.getByLabel('Email').fill(email) — Uses this.page (the stored browser reference) to type the email. The value comes from the caller, making the method flexible for any user.
  6. await this.page.getByRole('button', { name: 'Sign in' }).click() — Clicks Sign in to complete the login flow.
  7. Usage in a test: const loginPage = new LoginPage(page); then await loginPage.login('user@test.com', 'pass'); — The test is now two readable lines. If the login UI changes, you update only the LoginPage class.

Beginner Notes

  1. The Page Object Model (POM) pattern puts all locators and actions for one page into one class file.
  2. If a locator or label changes in the app, you fix it in one place — all tests that use the class are automatically fixed.
  3. Test files should read like plain English: "login, navigate to orders, assert order count" — not raw locator chains.

Common Mistake to Avoid

Mixing page locators and business assertions in one huge file. Split by responsibility.
