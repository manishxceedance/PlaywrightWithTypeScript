Module 9: Test Data

Beginner Friendly This module explains concepts in plain language and then gives practical examples.

What You Will Learn

How to separate data from logic and run data-driven tests.

Simple Explanation

Instead of copying test code for each user, keep one test logic and run it with multiple datasets.

Real-World Example

Login checks for admin, manager, and viewer can run from one loop using different credentials.

Step-by-Step Example (Code)

const users = [
  { role: 'admin', email: 'admin@example.com' },
  { role: 'viewer', email: 'viewer@example.com' }
];

for (const u of users) {
  test(`login ${u.role}`, async ({ page }) => {
    await page.getByLabel('Email').fill(u.email);
  });
}

Beginner Notes (Line by Line)

  1. const users = [ - This sets up a list of users to test with.
  2. { role: 'admin', email: 'admin@example.com' }, - This is a supporting line for the Test data example.
  3. { role: 'viewer', email: 'viewer@example.com' } - This is a supporting line for the Test data example.
  4. ]; - This is a supporting line for the Test data example.
  5. for (const u of users) { - This repeats the next steps for each item in a list.
  6. test(`login ${u.role}`, async ({ page }) => { - This line starts a new test scenario for Test data.
  7. await page.getByLabel('Email').fill(u.email); - This finds a text box by its label and types in the value.
  8. }); - This is a supporting line for the Test data example.
  9. } - This closes a block of code.

Common Mistake to Avoid

Hardcoding data in many places. Put data in fixtures/files for easier maintenance.

Practice Exercise

  1. Create login tests for three roles from one dataset.
  2. Add negative test data for invalid credentials.
