Module 17: Advanced Topics

Beginner Friendly This module explains concepts in plain language and then gives practical examples.

What You Will Learn

How to use interception, mocks, visual checks, retries, and tags.

Simple Explanation

Advanced features help control unstable dependencies and improve confidence in complex apps.

Real-World Example

If backend is unstable, mock API response so UI flow still validates expected frontend behavior.

Step-by-Step Example (Code)

await page.route('**/api/profile', route => route.fulfill({
  status: 200,
  body: JSON.stringify({ name: 'Mock User', plan: 'Pro' })
}));

await expect(page).toHaveScreenshot('dashboard.png');
test('checkout @smoke @regression', async ({ page }) => {});
Line-by-line explanation:
  1. await page.route('**/api/profile', route => route.fulfill({...})) — Intercepts every HTTP request whose URL matches the pattern and returns your custom fake response instead of calling the real server. The real backend is never contacted. Use this to test error states, loading spinners, or features that depend on backend data not yet built.
  2. status: 200, body: JSON.stringify({...}) — The fake HTTP response: a 200 OK with a JSON body. You can change this to status: 500 to simulate server errors and verify that your app handles them gracefully.
  3. await expect(page).toHaveScreenshot('dashboard.png') — Visual regression test. On the first run, Playwright saves a baseline screenshot. Every subsequent run compares the page pixel-by-pixel against the baseline. If they differ beyond the threshold, the test fails and shows a diff image.
  4. @smoke and @regression tags in the test name — Allow selective execution. Run only smoke tests with: npx playwright test --grep @smoke. Skip slow visual tests in PR checks with: npx playwright test --grep-invert @visual.
  5. Retries: Set retries: 1 in config for environments with occasional flakiness (network timeouts, CI slowness). Do not use retries to hide real bugs — always investigate root causes.

Beginner Notes

  1. API mocking lets you test UI behaviour in isolation from backend availability — tests become faster and more reliable.
  2. Visual regression testing catches unintended layout/style changes that functional assertions miss.
  3. Tags allow running specific subsets (smoke, regression, feature) without separate test files.

Common Mistake to Avoid

Using retries to hide real bugs. Use retries for transient issues while fixing root cause.

Practice Exercise

  1. Mock one unstable endpoint and validate UI behavior.
  2. Tag tests and run only smoke set.
