Module 5: Actions and Elements

Beginner Friendly This module explains concepts in plain language and then gives practical examples.

What You Will Learn

How to perform common UI actions safely, including alerts, iframes, and shadow DOM components.

Simple Explanation

Playwright can click, type, select dropdown values, check options, handle browser alerts, and work inside iframes and shadow DOM. These are common real-world blockers for beginners.

Real-World Example

A checkout flow usually needs typing address, selecting country, accepting terms, confirming an alert, and entering payment details inside iframe fields.

Step-by-Step Example (Code)

await page.getByRole('button', { name: 'Checkout' }).click();
      await page.getByLabel('Address').fill('Bengaluru');
      await page.locator('#country').selectOption('IN');
      await page.getByLabel('I agree').check();

      // Alert handling: register listener before clicking the alert trigger
      page.once('dialog', async dialog => {
        await expect(dialog.message()).toContain('Are you sure');
        await dialog.accept();
      });
      await page.getByRole('button', { name: 'Delete item' }).click();

      // Frame handling: switch to iframe with frameLocator
      const paymentFrame = page.frameLocator('#payment-frame');
      await paymentFrame.getByPlaceholder('1234 1234 1234 1234').fill('4242 4242 4242 4242');
      await paymentFrame.getByLabel('Expiry').fill('12/30');

      // Shadow DOM handling: Playwright supports open shadow roots through locator chaining
      await page.locator('user-card').locator('[data-testid="menu-button"]').click();
Line-by-line explanation:
  1. await page.getByRole('button', { name: 'Checkout' }).click() — Finds and clicks the "Checkout" button by its accessible name. Always prefer role-based locators for buttons and links.
  2. await page.getByLabel('Address').fill('Bengaluru') — Finds the input field associated with the "Address" label and types the value. Works reliably because it uses the HTML <label> element.
  3. await page.locator('#country').selectOption('IN') — Selects "IN" from a <select> dropdown by its option value. You can also select by visible text: selectOption('India').
  4. await page.getByLabel('I agree').check() — Ticks a checkbox. Use uncheck() to untick, or isChecked() to read the current state.
  5. page.once('dialog', async dialog => { ... }) — Registers a one-time listener for browser dialogs (alert/confirm/prompt). Critical: register this BEFORE clicking the button that triggers the dialog.
  6. await expect(dialog.message()).toContain('Are you sure') — Validates the dialog message text before accepting, ensuring the right dialog appeared.
  7. page.frameLocator('#payment-frame') — Creates a scoped reference to an iframe. All locator calls on this reference search only inside that frame. You cannot use page.locator() to reach elements inside iframes directly.
  8. paymentFrame.getByPlaceholder('1234 ...').fill('4242 ...') — Interacts with an iframe element exactly like a normal page element — after obtaining the frame reference.
  9. page.locator('user-card').locator('[data-testid="menu-button"]') — For open shadow DOM, chain from the host element (user-card) to an inner element. Playwright automatically pierces open shadow roots when chaining locators.

Beginner Notes

  1. Always register page.once('dialog', ...) before clicking the button that opens the dialog.
  2. Use frameLocator() for iframe content — normal page.locator() cannot reach inside iframes.
  3. For shadow DOM, chain locators from the host custom element to the internal element.

How to think about these actions:
  1. click() is the default action, but you should still wait for the UI state that proves the click had the intended result.
  2. fill() replaces the current value in one shot. Use it for most form fields instead of typing character by character.
  3. clear() is useful for search boxes and reusable inputs where the page already contains text from a previous step.
  4. press() matters when the application reacts to keyboard behavior, such as pressing Enter to submit a dialog or Tab to move focus.
  5. selectOption() is only for native <select> elements. Custom dropdowns usually need normal clicks on their visible options.
  6. check() and uncheck() are safer than clicking checkboxes directly because they express the target state instead of just toggling.
  7. setInputFiles() targets the hidden native file input, not the visual upload card around it.
  8. Dialogs, downloads, and popups are event-based actions. Register the wait or listener before the click that triggers them.
  9. frameLocator() and shadow DOM locator chaining are not special actions by themselves. They are the path that lets normal actions reach otherwise isolated content.
  10. Tables and modals are where real-world tests become flaky. Scope actions to one row or one dialog before clicking or typing.

What the Sample Apps Cover Well

  • Form actions: text entry, date/time entry, select, multi-select, checkbox, radio, search, textarea, reset.
  • File-based actions: image upload and resume upload.
  • Event-based actions: alert, confirm, prompt, popup, download, delayed dialog.
  • Complex surfaces: iframe, shadow DOM, sortable tables, row-scoped controls, and modal dialogs.

What This Repo Does Not Explicitly Demonstrate Yet

The current sample apps do not provide a strong dedicated lab for actions like dragTo(), right-click context menus, double-click-only behavior, or editable rich-text regions. Those are valid Playwright actions, but they are not grounded in these sample pages yet, so this module focuses on the actions you can genuinely practice here.

Common Mistake to Avoid

Trying to interact with iframe elements directly from page or adding dialog listener after click. Register dialog handler first, then trigger action.
