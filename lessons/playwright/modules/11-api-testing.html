Module 11: API Testing

Beginner Friendly This module explains concepts in plain language and then gives practical examples.

What You Will Learn

How to call APIs directly and validate responses.

Simple Explanation

Playwright can test API endpoints without opening UI. This speeds up backend checks and supports setup/teardown data creation.

Real-World Example

Before UI checkout test, create test order through API so UI scenario starts in the right state.

Step-by-Step Example (Code)

test('create booking', async ({ request }) => {
  const response = await request.post('/api/bookings', {
    data: { name: 'Asha', room: 'A101' }
  });
  expect(response.ok()).toBeTruthy();
  const body = await response.json();
  expect(body.name).toBe('Asha');
});

Beginner Notes (Line by Line)

  1. test('create booking', async ({ request }) => { - This line starts a new test scenario for Api testing.
  2. const response = await request.post('/api/bookings', { - This sends a direct API request to the server, skipping the UI.
  3. data: { name: 'Asha', room: 'A101' } - This provides the data to send in an API request.
  4. }); - This is a supporting line for the Api testing example.
  5. expect(response.ok()).toBeTruthy(); - This checks if what happened matches what you expected.
  6. const body = await response.json(); - This gets the details from the server's response.
  7. expect(body.name).toBe('Asha'); - This checks if what happened matches what you expected.
  8. }); - This is a supporting line for the Api testing example.

Common Mistake to Avoid

Checking only status code. Also validate key fields in response body.

Practice Exercise

  1. Write one GET and one POST API test.
  2. Validate status, response fields, and value types.
