Module 7: Browser Handling

Beginner Friendly This module explains concepts in plain language and then gives practical examples.

What You Will Learn

How to manage tabs, contexts, cookies, and sessions.

Simple Explanation

Browser context is like an isolated browser profile. Multiple contexts let you test admin and user behavior in parallel.

Real-World Example

You can keep an admin logged in one context and a customer in another, then verify both sides of a workflow.

Step-by-Step Example (Code)

const newTabPromise = page.context().waitForEvent('page');
await page.getByRole('link', { name: 'Open docs' }).click();
const newTab = await newTabPromise;
const cookies = await page.context().cookies();
Line-by-line explanation:
  1. const newTabPromise = page.context().waitForEvent('page') — Creates a listener that will capture the next new browser tab to open. Important: set this up BEFORE clicking the link that opens the new tab, otherwise the tab can open and be missed.
  2. await page.getByRole('link', { name: 'Open docs' }).click() — Clicks the "Open docs" link. This action causes a new browser tab to open.
  3. const newTab = await newTabPromise — Waits for the new tab to appear and stores a reference to it. You can now use newTab.url(), newTab.title(), or any other page methods on the new tab.
  4. await page.context().cookies() — Reads all cookies in the current browser context. Useful to verify session tokens, persistent login cookies, or feature-flag cookies without inspecting DevTools manually.
  5. Browser context isolation: Each BrowserContext is like an independent incognito profile with its own cookies, localStorage, and sessions. Use separate contexts to test admin and regular user roles in the same test run without interference.

Beginner Notes

  1. Always call waitForEvent('page') before the action that opens a new tab.
  2. Each browser context is isolated — like a separate incognito window.
  3. Use two contexts to test interactions between two different user roles in one test.

Common Mistake to Avoid

Using one shared session for all tests, which can cause hidden cross-test dependency.

Practice Exercise

  1. Capture a new tab and validate URL/title.
  2. Create two contexts and run role-based checks.
