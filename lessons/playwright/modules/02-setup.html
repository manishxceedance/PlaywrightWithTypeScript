Module 2: Setup

Beginner Friendly Learn installation, first run, and the most important Playwright config options.

What You Will Learn

How to install prerequisites, scaffold a Playwright project, and configure basic quality settings.

Simple Explanation

Setup is where you prepare tools once so test execution becomes easy every day. Good setup reduces flaky tests and onboarding time.

Step-by-Step Commands

node -v
npm -v
mkdir playwright-lab
cd playwright-lab
npm init -y
npm init playwright@latest
npx playwright test
What each command does:
  1. node -v / npm -v — Verifies Node.js and npm are installed. Playwright requires Node.js 18 or above.
  2. mkdir playwright-lab — Creates a new project folder. All your test files will live here.
  3. cd playwright-lab — Moves your terminal into the new folder.
  4. npm init -y — Creates a package.json (project manifest). The -y flag accepts all default settings automatically.
  5. npm init playwright@latest — Downloads Playwright, installs the three browsers (Chromium, Firefox, WebKit), and generates a starter playwright.config.ts and example test files.
  6. npx playwright test — Runs every test file in the project. Results appear in the terminal and in a generated HTML report.

playwright.config.ts Essentials

  • baseURL for shorter navigation code.
  • retries for transient CI instability.
  • reporter for readable results and trace links.
  • projects for cross-browser runs.

Common Mistake to Avoid

Running default config forever. Always customize at least base URL, retries, and reporting before framework growth.

Practice Exercise

  1. Create a fresh Playwright project and run sample tests.
  2. Set baseURL and use relative navigation in one test.
  3. Enable HTML report and open it after execution.
