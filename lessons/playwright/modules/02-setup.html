    What You Will Learn

    How to install tools, run your first test, and understand Playwright configuration.

    Simple Explanation

    You need Node.js to run JavaScript/TypeScript tools. Playwright setup creates a ready test project with config and sample tests. Configuration controls where tests run, timeouts, retries, reports, and base URL.

    Real-World Example

    Like installing kitchen tools before cooking, setup prepares everything once so daily testing becomes easy.

    Playwright Configuration (Most Important Options)

    After setup, the next thing every beginner should learn is playwright.config.ts. This file is the control center of your framework.

    import { defineConfig, devices } from '@playwright/test';
        
    Running commands outside project folder or never customizing playwright.config.ts. Always set at least baseURL, reporter, and failure artifacts.

    Config Notes (Line by Line)
    
          
      
            
    1. testDir: './tests' - Tells Playwright where your test files are located.
      2. 
            
    2. timeout: 30_000 - Maximum time allowed for one test before it fails.
      3. 
            
    3. expect.timeout: 5_000 - Maximum wait time for assertion conditions like toBeVisible().
      4. 
            
    4. retries: 1 - If a test fails, Playwright tries one extra time before final failure.
      5. 
            
    5. reporter: [['html'], ['list']] - Generates readable terminal output and HTML report.
      6. 
            
    6. baseURL - Lets you write page.goto('/login') instead of full URL everywhere.
      7. 
            
    7. screenshot/video/trace - Captures useful artifacts for debugging failed tests.
      8. 
            
    8. projects - Runs the same suite across multiple browsers with one command.
