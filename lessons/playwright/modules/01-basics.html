Module 1: Basics

Beginner Friendly Playwright Track

This module explains concepts in plain language and then gives practical examples. Automation means writing repeatable instructions for a computer to test your app. It saves time on repeated checks and reduces human mistakes.

What You Will Learn

What automation means, why teams use it, and where Playwright fits.

Simple Explanation

Automation means writing repeatable instructions for a computer to test your app. It saves time on repeated checks and reduces human mistakes.

Real-World Example

If your login flow must be tested daily, automation runs the same steps every day and reports pass or fail.

Step-by-Step Example (Code)

// Basic Playwright smoke test
import { test, expect } from '@playwright/test';

test('homepage loads', async ({ page }) => {
  await page.goto('https://example.com');
  await expect(page).toHaveTitle(/Example/);
});

Beginner Notes (Line by Line)

  1. // Basic Playwright smoke test - This is a comment describing the next step.
  2. import { test, expect } from '@playwright/test'; - This line brings in Playwright's test tools so you can write and run tests.
  3. test('homepage loads', async ({ page }) => { - This line starts a new test scenario for Basics.
  4. await page.goto('https://example.com'); - This opens the web page you want to test.
  5. await expect(page).toHaveTitle(/Example/); - This checks if what happened matches what you expected.
  6. }); - This is a supporting line for the Basics example.

Common Mistake to Avoid

Thinking automation replaces all manual testing. It supports manual testing, not replaces it.
