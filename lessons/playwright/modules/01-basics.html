Module 1: Basics

Beginner Friendly Start here to understand why automation exists and how Playwright fits into manual testing work.

What You Will Learn

What automation means, where Playwright helps, and what a simple first test looks like.

Simple Explanation

Automation means writing repeatable steps once, then running them many times consistently. It reduces repetitive manual effort and catches regressions faster.

Real-World Example

If your team checks login every day, a Playwright smoke test can open app, log in, and verify dashboard in minutes.

Step-by-Step Example (Code)

import { test, expect } from '@playwright/test';

test('homepage loads', async ({ page }) => {
  await page.goto('https://example.com');
  await expect(page).toHaveTitle(/Example/);
});
Line-by-line explanation:
  1. import { test, expect } from '@playwright/test'; — Imports Playwright's test runner and assertion library. Every test file starts with this line.
  2. test('homepage loads', async ({ page }) => { — Defines one test scenario. The text in quotes is the name shown in reports. async ({ page }) gives you a page object that represents a browser tab.
  3. await page.goto('https://example.com'); — Opens the URL in the browser. await pauses execution until the page finishes loading before moving to the next line.
  4. await expect(page).toHaveTitle(/Example/); — Checks that the browser tab title matches the pattern. Playwright auto-retries this assertion until it passes or the timeout is reached. If it never matches, the test fails with a descriptive error message.

Beginner Notes

  1. A single test() block = one automated scenario.
  2. page.goto() navigates the browser to a URL — like typing in the address bar.
  3. expect() validates that the actual state of the page matches what you expected.

Common Mistake to Avoid

Thinking automation replaces all manual testing. It supports manual testing by covering repeatable checks.

Practice Exercise

  1. Write three repetitive checks from your project that can be automated first.
  2. Write one sentence on what pass/fail means in automation.
