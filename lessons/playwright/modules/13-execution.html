Module 13: Execution

Beginner Friendly This module explains concepts in plain language and then gives practical examples.

What You Will Learn

How to run tests in parallel and across browsers.

Simple Explanation

Parallel workers reduce total runtime. Browser projects help verify behavior on multiple engines.

Real-World Example

A flow works in Chromium but fails in WebKit due to rendering difference. Cross-browser run finds this early.

Step-by-Step Example (Code)

// playwright.config.ts
projects: [
  { name: 'chromium', use: { browserName: 'chromium' } },
  { name: 'firefox', use: { browserName: 'firefox' } },
  { name: 'webkit', use: { browserName: 'webkit' } }
];
Line-by-line explanation:
  1. projects: [...] — Defines multiple browser and device configurations. Playwright runs all projects automatically, so your entire test suite executes once per browser entry.
  2. { name: 'chromium', use: { browserName: 'chromium' } } — One project for Chromium (the engine behind Chrome and Edge). The name appears in reports to show which browser each result came from.
  3. { name: 'firefox', ... } and { name: 'webkit', ... } — Same pattern for Firefox and Safari (WebKit). Adding these catches browser-specific bugs that would be invisible in Chromium-only testing.
  4. Run one browser only: npx playwright test --project=chromium — useful during development to run fast and switch to full cross-browser only in CI.
  5. Parallel workers: Set workers: 4 in playwright.config.ts to run 4 tests simultaneously. Each worker gets its own browser context so tests are fully isolated.

Beginner Notes

  1. Parallel execution runs multiple tests at the same time, dramatically reducing total suite runtime.
  2. Cross-browser testing reveals rendering differences and browser-specific JavaScript bugs early.
  3. Start with Chromium only during development, then run all browsers in CI before release.

Common Mistake to Avoid

Setting too many workers for unstable test environment. Tune worker count to infrastructure capacity.

Practice Exercise

  1. Run suite with 2 workers and compare runtime.
  2. Run same test in Chromium and Firefox and compare behavior.
