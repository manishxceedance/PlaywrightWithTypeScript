Final Project Rubric: Playwright + TypeScript

Assessment: Practical Use this rubric to evaluate if you can automate a new web application independently.

How to Use This Rubric

  1. Build one mini framework project using your own application under test.
  2. Score each category from 0 to 5.
  3. Total your score and map your level.

Scoring Scale

  • 0: Not done
  • 1: Attempted with heavy help
  • 2: Works with frequent guidance
  • 3: Independent but inconsistent
  • 4: Independent and stable
  • 5: Team-ready and can mentor others

Category Rubric (0-5 each)

  1. Framework structure: clean folder design, POM layering, reusable helpers/fixtures.
  2. TypeScript quality: strict typing, interfaces/types, no avoidable any usage.
  3. Locator strategy: semantic locators, row/section scoping, low flakiness.
  4. Assertions: business-focused, meaningful failure messages, negative checks where needed.
  5. Data strategy: test data separated from logic, environment-safe credentials.
  6. Execution strategy: smoke/regression tags, cross-browser setup, worker tuning.
  7. Debugging discipline: trace/report/screenshot usage and root-cause documentation.
  8. CI readiness: pipeline execution, artifact upload, stable quality gate.

Score Bands (Out of 40)

  • 0-15: Beginner
  • 16-24: Intermediate
  • 25-32: Advanced
  • 33-40: Expert

Definition of Expert (Practical)

  • Can automate a new application without template copy-paste dependency.
  • Can keep suite stable while UI evolves.
  • Can triage failures quickly and communicate root cause clearly.
  • Can maintain TypeScript framework quality in team code reviews.
