REST API Documentation

🔌 HRMS REST API

A fully functional REST API running via Service Worker — no server required. All requests are intercepted and processed locally with IndexedDB persistence. Use this page to explore endpoints, read request/response schemas, and try live calls directly in the browser.

Base URL: ./hrms-api

🔐 Authentication

Most write operations require a Bearer token. Obtain one via POST /auth/login. Include it as: Authorization: Bearer <token>

Authentication
POST /auth/login Authenticate user, get Bearer token

Returns a session token and user object. Use the token in the Authorization header for protected endpoints.

Request Body:
{ "username": "admin", "password": "admin" }
Response 200:
{ "token": "tk_abc123...", "user": { "id": "USR-0000", "username": "admin", "firstName": "Admin", "role": "admin" } }
POST /auth/register Register new user (max 100 stored)
Required fields:
{ "username": "jsmith", "password": "securePass123", "email": "j.smith@company.com", "firstName": "John", "lastName": "Smith" }
Response 201:
{ "id": "USR-0002", "username": "jsmith", "role": "employee" }
GET /auth/me Get current authenticated user

Requires Authorization: Bearer <token> header.

POST /auth/logout Invalidate current session token

Requires Bearer token. Returns {"success":true}.

Employees
GET /employees List employees with pagination & filters
Query Parameters: ${paramRow('page', 'number', '1', 'Page number')} ${paramRow('limit', 'number', '20', 'Results per page (max 100)')} ${paramRow('search', 'string', '—', 'Full-text search: name, email, ID, title')} ${paramRow('department', 'string', '—', 'Filter by department')} ${paramRow('status', 'string', '—', 'Active | Inactive | On Leave')} ${paramRow('sort', 'string', 'firstName', 'Sort field')} ${paramRow('order', 'string', 'asc', 'asc | desc')}
Param Type Default Description
Response:
{ "data": [ { ...employee } ], "pagination": { "page": 1, "limit": 20, "total": 400, "pages": 20 } }
POST /employees Create new employee 🔒

🔒 Requires Bearer token.

Required fields:
{ "firstName": "Jane", "lastName": "Doe", "email": "jane.doe@company.com", "department": "Engineering", "jobTitle": "Frontend Developer", "salary": 85000, "hireDate": "2024-01-15", "status": "Active", "employmentType": "Full-time" }
GET /employees/{id} Get single employee by ID
PUT /employees/{id} Update employee 🔒

🔒 Requires Bearer token. Send only the fields you want to update.

{ "status": "Inactive", "salary": 90000 }
DELETE /employees/{id} Delete employee 🔒

🔒 Requires Bearer token. Deletes the employee permanently from IndexedDB.

GET /employees/export Download all employees as CSV

Returns a CSV file download with all employee records.

Meta
GET /stats Dashboard statistics
GET /departments List all departments
GET /users List registered users 🔒 Admin only

🔒 Requires admin Bearer token.
