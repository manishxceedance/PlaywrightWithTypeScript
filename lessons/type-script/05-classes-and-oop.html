Lesson 5: Classes and OOP (Object-Oriented Programming)

Level: Intermediate Learn how to build reusable, organized code using TypeScript classes and OOP features.

Classes help you build reusable framework components like page objects, API clients, and test reporters. OOP lets you organize code for real-world automation and apps.

1. What is a Class?

A class is a blueprint for creating objects with shared structure and behavior. Use classes for things that have properties and actions.

// A simple test suite class
class TestSuite {
  name: string;
  totalCases: number;

  constructor(name: string, totalCases: number) {
    this.name = name;
    this.totalCases = totalCases;
  }

  getSummary(): string {
    return `${this.name}: ${this.totalCases} cases`;
  }
}

const suite = new TestSuite("Login", 12);
console.log(suite.getSummary()); // Login: 12 cases

2. Access Modifiers

Access modifiers control where a property or method can be used.

  • public: accessible everywhere (default).
  • private: only inside this class.
  • protected: inside this class and subclasses.
class User {
  public name: string;
  private password: string;
  protected role: string;

  constructor(name: string, password: string, role: string) {
    this.name = name;
    this.password = password;
    this.role = role;
  }
}

3. Inheritance and super()

One class can extend another to reuse and customize behavior. Use super() to call the parent’s constructor or methods.

class BasePage {
  navigate(url: string): void {
    console.log(`Open ${url}`);
  }
}

class LoginPage extends BasePage {
  login(user: string, pass: string): void {
    console.log(`Login with ${user}`);
  }
}

const page = new LoginPage();
page.navigate("/login");
page.login("alice", "secret");

4. Static Members

static properties and methods belong to the class itself, not to instances. Use them for helpers or shared values.

class MathUtils {
  static PI = 3.14;
  static square(x: number) {
    return x * x;
  }
}
console.log(MathUtils.PI);
console.log(MathUtils.square(5));

5. Abstract Classes

An abstract class can’t be created directly. It defines a contract for subclasses to implement.

abstract class Reporter {
  abstract report(message: string): void;
  log(msg: string) {
    console.log("LOG:", msg);
  }
}

class ConsoleReporter extends Reporter {
  report(message: string): void {
    console.log(`[REPORT] ${message}`);
  }
}

const rep = new ConsoleReporter();
rep.report("All tests passed");

6. Interface + Class Contract

Classes can implement interfaces to guarantee they provide certain methods or properties.

interface Logger {
  log(message: string): void;
}

class FileLogger implements Logger {
  log(message: string): void {
    // Write to file (simulated)
    console.log(`[FILE] ${message}`);
  }
}

7. Real-World Example: API Client

Combine these features to build a real-world API client class.


            class ApiClient {
  private baseUrl: string;
  constructor(baseUrl: string) {
    this.baseUrl = baseUrl;
  }
  get(endpoint: string) {
    console.log(`GET ${this.baseUrl}${endpoint}`);
  }
}

class SecuredApiClient extends ApiClient {
  private token: string;
  constructor(baseUrl: string, token: string) {
    super(baseUrl);
    this.token = token;
  }
  get(endpoint: string) {
    console.log(`GET ${endpoint} with token ${this.token}`);
  }
}
const api = new SecuredApiClient("https://api.example.com", "my-token");
api.get("/users");
 
const api2 = new ApiClient("https://api.example.com");
api2.get("/users");
