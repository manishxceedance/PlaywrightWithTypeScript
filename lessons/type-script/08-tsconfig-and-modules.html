Lesson 8: tsconfig, Modules, and Project Quality

Level: Advanced Learn how to configure, organize, and scale TypeScript projects for teams.

This lesson teaches how to set up your project for long-term quality, collaboration, and maintainability.

1. Key tsconfig Settings

Your tsconfig.json controls how TypeScript checks and compiles your code. Here’s a safe starting point:

{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "rootDir": "src",
    "outDir": "dist",
    "strict": true,
    "noUnusedLocals": true,
    "noImplicitReturns": true
  },
  "include": ["src"]
}
Explanation: This configures TypeScript to use modern JS features, output files to dist, and enforce strict type checks. strict: true enables many safety checks, and noUnusedLocals/noImplicitReturns catch common mistakes.

2. Why strict mode matters

strict: true enables a group of checks that catch risky mistakes. For testing code, this helps avoid hidden null issues and incorrect payload handling.

3. ES Modules: Export and Import

Use export and import to split code into reusable modules.

// src/utils/math.ts
export function passRate(passed: number, total: number): number {
  return total === 0 ? 0 : (passed / total) * 100;
}

// src/index.ts
import { passRate } from "./utils/math";
console.log(passRate(47, 50));
Explanation: export makes a function available to other files. import brings it in for use. This modular approach keeps code organized and reusable.

4. Module Patterns

Organize code by feature (e.g., utils, services, models). Use one file per type or interface for clarity.

5. Suggested Scripts for Team Projects

Automate common tasks with npm scripts.

{
  "scripts": {
    "build": "tsc -p tsconfig.json",
    "clean": "rimraf dist",
    "typecheck": "tsc --noEmit"
  }
}

6. Common Utility Types

TypeScript comes with built-in utility types that help you transform and work with types more easily. Here are the most useful ones for beginners:

  • Partial<T>: Makes all properties in a type optional. 
    interface Bug {
  id: string;
  summary: string;
  severity: string;
}
// Partial means all fields are optional
const draft: Partial = { summary: "Crash on login" };
    Explanation: Partial<T> makes every property in a type optional. Here, draft can have any subset of Bug's fields.
  • Pick<T, K>: Lets you create a type with only some properties from another type. 
    // Only id and summary from Bug
type BugSummary = Pick;
const summary: BugSummary = { id: "B-1", summary: "Crash" };
    Explanation: Pick<T, K> creates a new type with only the specified properties from the original type. BugSummary has just id and summary.
  • Omit<T, K>: Like Pick, but removes properties. 
    // All Bug fields except severity
type BugNoSeverity = Omit;
const bug: BugNoSeverity = { id: "B-2", summary: "Slow" };
    Explanation: Omit<T, K> creates a type with all properties except the ones you list. BugNoSeverity has id and summary only.
  • Record<K, T>: Makes an object type with keys of type K and values of type T. 
    // Map test case IDs to results
type TestResults = Record;
const results: TestResults = { "TC-1": "pass", "TC-2": "fail" };
    Explanation: Record<K, T> creates an object type where keys are of type K and values are of type T. Here, results maps test case IDs to their result.

Tip: Utility types help you avoid repeating yourself and make your code safer and more flexible.

7. Real Project Checklist

  • Enable strict mode and keep it on.
  • Use clear module boundaries (utils, services, models).
  • Use separate files for types/interfaces.
  • Run typecheck command in CI pipeline.
