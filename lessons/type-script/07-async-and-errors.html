Lesson 7: Async Programming and Error Handling

Level: Advanced Learn how to write reliable async code and handle errors in TypeScript.

Most test automation and API tasks are asynchronous. Typed async code avoids fragile workflows and makes your code robust.

1. What is Asynchronous Code?

Async code lets you do things that take time (like API calls or timers) without blocking the rest of your program.

2. Promise Basics

A Promise represents a value that will be available later. Use Promise for async operations.

function waitMs(ms: number): Promise {
  return new Promise((resolve) => {
    setTimeout(() => resolve(`Waited ${ms} ms`), ms);
  });
}
Explanation: This function returns a Promise that resolves after a given number of milliseconds. It uses setTimeout to wait, then calls resolve with a message. This is a basic pattern for creating asynchronous delays.

3. async/await

async functions always return a Promise. Use await to pause until a Promise is ready.

async function runHealthCheck(): Promise {
  const message = await waitMs(300);
  console.log(message);
}
runHealthCheck();
Explanation: The async keyword makes the function always return a Promise. Inside, await pauses execution until waitMs(300) resolves, then logs the result. This makes async code look and behave more like synchronous code.

4. Typed API Fetch Example

Always type your async results for safety.

type HealthResponse = { status: string; uptime: number };

async function fetchHealth(): Promise {
  const response = await fetch("/health");
  if (!response.ok) {
    throw new Error(`HTTP ${response.status}`);
  }
  return (await response.json()) as HealthResponse;
}
Explanation: This function fetches data from /health and expects a JSON response matching the HealthResponse type. If the HTTP response is not OK, it throws an error. This ensures you always get a correctly-typed result or a clear error.

5. Error Handling Pattern

Use try/catch to handle errors in async code. Always check the error type.

async function safeHealthCheck(): Promise {
  try {
    const health = await fetchHealth();
    console.log("Service status:", health.status);
  } catch (error) {
    if (error instanceof Error) {
      console.error("Health check failed:", error.message);
    } else {
      console.error("Unknown failure", error);
    }
  }
}
Explanation: This uses try/catch to handle errors from async code. It checks if the error is an instance of Error for safe logging. This pattern prevents crashes and gives meaningful error messages.

6. Parallel Async: Promise.all

Use Promise.all to run multiple async tasks at once and wait for all results.

async function runAll() {
  const [a, b] = await Promise.all([
    waitMs(100),
    waitMs(200)
  ]);
  console.log(a, b);
}
runAll();
Explanation: Promise.all runs multiple async tasks in parallel and waits for all to finish. The results are returned as an array. This is efficient for independent async operations.

7. Utility: Safe Fetch Helper

Write a generic async helper for safe API calls.

async function safeFetchJson(url: string): Promise {
  try {
    const res = await fetch(url);
    if (!res.ok) throw new Error("Bad response");
    return await res.json();
  } catch (e) {
    return null;
  }
}
Explanation: This generic helper fetches JSON from a URL and returns either the typed result or null if anything fails. It’s a reusable pattern for safe API calls.
