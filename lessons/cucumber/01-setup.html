Lesson 1: Project Setup — Cucumber + Playwright + TypeScript

Level: Beginner Follow every step in order. Each command must succeed before continuing.

1. Prerequisites

Before you begin, make sure these are installed:

  • Node.js 18+ — run node -v to verify.
  • npm 9+ — run npm -v to verify.
  • VS Code — recommended editor.
  • Cucumber (Gherkin) Full Support VS Code extension — for syntax highlighting in .feature files.

2. Create Project Folder

mkdir cucumber-playwright-bdd
cd cucumber-playwright-bdd
npm init -y

3. Install Dependencies

Install all required packages in one command:

npm install --save-dev \
  @cucumber/cucumber \
  @playwright/test \
  playwright \
  typescript \
  ts-node \
  @types/node

Then install Playwright browser binaries:

npx playwright install chromium

What each package does:

  • @cucumber/cucumber — the core Cucumber runner that reads .feature files.
  • @playwright/test — Playwright browser automation library.
  • playwright — the browser binaries installer.
  • typescript — compiles TypeScript to JavaScript.
  • ts-node — runs TypeScript files directly without a separate compile step.
  • @types/node — TypeScript type definitions for Node.js APIs.

4. TypeScript Configuration

Create tsconfig.json in the project root:

{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "lib": ["ES2020"],
    "strict": true,
    "esModuleInterop": true,
    "resolveJsonModule": true,
    "outDir": "dist",
    "rootDir": "src",
    "baseUrl": ".",
    "paths": {
      "@pages/*": ["src/pages/*"],
      "@support/*": ["src/support/*"]
    }
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}

5. Cucumber Configuration

Create cucumber.json in the project root. This tells Cucumber where to find feature files and step definitions:

{
  "default": {
    "require": ["src/steps/**/*.ts", "src/support/**/*.ts"],
    "requireModule": ["ts-node/register"],
    "format": [
      "progress-bar",
      "html:reports/cucumber-report.html",
      "json:reports/cucumber-report.json"
    ],
    "paths": ["src/features/**/*.feature"],
    "parallel": 1
  }
}

6. Recommended Folder Structure

cucumber-playwright-bdd/
  src/
    features/          ← .feature files (Gherkin scenarios)
      login.feature
      checkout.feature
    steps/             ← Step definition TypeScript files
      login.steps.ts
      checkout.steps.ts
    support/           ← Hooks, world, shared setup
      hooks.ts
      world.ts
    pages/             ← Page Object Model classes
      LoginPage.ts
      CheckoutPage.ts
  reports/             ← Generated HTML/JSON reports (auto-created)
  cucumber.json        ← Cucumber runner configuration
  tsconfig.json        ← TypeScript configuration
  package.json

Create the folder structure with these commands (PowerShell on Windows):

New-Item -ItemType Directory src/features, src/steps, src/support, src/pages, reports

7. Add npm Scripts to package.json

Open package.json and replace the scripts section:

"scripts": {
  "test": "cucumber-js",
  "test:headed": "HEADED=true cucumber-js",
  "test:tags": "cucumber-js --tags",
  "report": "open reports/cucumber-report.html"
}

On Windows use start instead of open for the report script.

8. Verify the Setup With a Smoke Test

Create your first (empty) feature file at src/features/smoke.feature:

Feature: Smoke test

  Scenario: Setup verification
    Given the project is configured correctly
    When I run the test suite
    Then no configuration errors should appear

Create the matching step definition at src/steps/smoke.steps.ts:

import { Given, When, Then } from '@cucumber/cucumber';

Given('the project is configured correctly', function () {
  // intentionally empty — verifies wiring only
});

When('I run the test suite', function () {
  // intentionally empty
});

Then('no configuration errors should appear', function () {
  // intentionally empty
});

Run the smoke test:

npm test

Expected output:

1 scenario (1 passed)
3 steps (3 passed)
0m00.012s

9. Common Setup Problems and Fixes

  • Cannot find module 'ts-node/register' — run npm install --save-dev ts-node again.
  • Feature file not found — check that the paths value in cucumber.json matches your actual folder name.
  • Step definition not matched — check that the require glob in cucumber.json points to where your .steps.ts files live.
  • Playwright browsers not installed — run npx playwright install.
  • SyntaxError: Cannot use import statement — your tsconfig.json must have "module": "commonjs".

Practice Exercise

  1. Follow every step above and create the project from scratch.
  2. Run npm test and confirm 1 scenario (1 passed).
  3. Add a second scenario to smoke.feature with a different Given/When/Then and create matching empty step definitions. Run again.
  4. Deliberately break the path in cucumber.json (e.g., change src/features to src/feat), run the suite, and read the error message. Then fix it.
