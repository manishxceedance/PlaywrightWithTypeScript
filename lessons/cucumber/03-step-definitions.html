Lesson 3: Step Definitions — Linking Gherkin to TypeScript

Level: Beginner → Intermediate This is the glue between plain-English scenarios and runnable automation code.

1. What Is a Step Definition?

A step definition is a TypeScript function decorated with Given, When, or Then. When Cucumber reads a step in a feature file, it looks for a matching step definition to execute. 

// Gherkin step:
// Given the user is on the login page

// Matching step definition:
import { Given } from '@cucumber/cucumber';

Given('the user is on the login page', async function () {
  await this.page.goto('https://manishxceedance.github.io/PlaywrightWithTypeScript/lessons/practice-app/hrms/login.html');
});

2. Importing Given, When, Then

All step decorators come from @cucumber/cucumber:

import { Given, When, Then, And, But } from '@cucumber/cucumber';

And and But are aliases for the previous keyword — you can use Given/When/Then for all steps if you prefer, Cucumber does not differentiate.

3. String Parameters

Text in double quotes in Gherkin is passed as a string parameter. Use curly-brace expressions in the step pattern to capture them: 

# Feature file
When the user enters username "admin" and password "secret"
// Step definition
import { When } from '@cucumber/cucumber';

When('the user enters username {string} and password {string}',
  async function (username: string, password: string) {
    await this.page.fill('#username', username);
    await this.page.fill('#password', password);
  }
);

Built-in Cucumber expression types:

  • {string} — captures quoted text (removes the quotes).
  • {int} — captures a whole number.
  • {float} — captures a decimal number.
  • {word} — captures a single unquoted word.

4. Using Regular Expressions

You can also use a RegExp instead of a Cucumber expression string. This gives more flexibility for complex patterns: 

When(/^the user enters "([^"]*)" in the search box$/, async function (query: string) {
  await this.page.fill('[data-testid="search-input"]', query);
});

5. The World Object (this)

Inside a step definition, this refers to the World — a shared context object that lives for the duration of one scenario. You use it to share state between steps (like a browser page object or test data). 

// src/support/world.ts
import { setWorldConstructor, World, IWorldOptions } from '@cucumber/cucumber';
import { Browser, BrowserContext, Page, chromium } from '@playwright/test';

export interface CustomWorld extends World {
  browser: Browser;
  context: BrowserContext;
  page: Page;
}

class PlaywrightWorld extends World implements CustomWorld {
  browser!: Browser;
  context!: BrowserContext;
  page!: Page;

  constructor(options: IWorldOptions) {
    super(options);
  }
}

setWorldConstructor(PlaywrightWorld);

Now every step in a scenario can read and write this.page, this.browser, etc.

6. Full Step Definition Example

// src/steps/login.steps.ts
import { Given, When, Then } from '@cucumber/cucumber';
import { expect } from '@playwright/test';
import { CustomWorld } from '../support/world';

Given('the user is on the login page', async function (this: CustomWorld) {
  await this.page.goto('https://manishxceedance.github.io/PlaywrightWithTypeScript/lessons/practice-app/hrms/login.html');
});

When(
  'the user enters username {string} and password {string}',
  async function (this: CustomWorld, username: string, password: string) {
    await this.page.fill('#username', username);
    await this.page.fill('#password', password);
  }
);

When('the user clicks the Login button', async function (this: CustomWorld) {
  await this.page.click('button[type="submit"]');
});

Then(
  'the user should be redirected to the dashboard',
  async function (this: CustomWorld) {
    await expect(this.page).toHaveURL(/hrms/);
  }
);

Then(
  'the welcome message should contain {string}',
  async function (this: CustomWorld, expectedText: string) {
    const heading = this.page.locator('#welcome-msg');
    await expect(heading).toContainText(expectedText);
  }
);

7. Pending Steps

When you create a scenario but haven't written the code yet, return 'pending' from the step. Cucumber reports it as pending (not a failure) which is useful during development: 

Then('the user sees the order confirmation page', function () {
  return 'pending';
});

8. Async vs Sync Steps

All Playwright operations are asynchronous. Always use async function and await for any step that touches the browser: 

// Correct — async with await
When('the user clicks Submit', async function (this: CustomWorld) {
  await this.page.click('button[type="submit"]');
});

// Wrong — missing await causes flaky tests
When('the user clicks Submit', function (this: CustomWorld) {
  this.page.click('button[type="submit"]'); // no await!
});

9. Sharing Data Between Steps

Use the World object to pass data between steps within the same scenario: 

// src/support/world.ts (extend the class)
class PlaywrightWorld extends World {
  browser!: Browser;
  context!: BrowserContext;
  page!: Page;
  orderId?: string;  // ← custom shared data
}
// In step definitions
When('the user places an order', async function (this: CustomWorld) {
  await this.page.click('#place-order');
  this.orderId = await this.page.locator('#order-id').textContent() ?? '';
});

Then('the order history should show the new order', async function (this: CustomWorld) {
  const row = this.page.locator(`tr:has-text("${this.orderId}")`);
  await expect(row).toBeVisible();
});

10. Step Definition Anti-Patterns to Avoid

  • Hard-coded waits — never use await new Promise(r => setTimeout(r, 3000)). Use Playwright's built-in auto-wait.
  • Duplicate step patterns — two steps with the same text cause an ambiguous step error. Keep patterns unique.
  • UI logic in feature files — selectors and DOM details belong in step definitions (or page objects), not in Gherkin text.
  • Not using await — every Playwright call returns a Promise; skipping await gives misleading passing tests.
