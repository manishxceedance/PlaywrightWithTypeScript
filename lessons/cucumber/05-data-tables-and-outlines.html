Lesson 5: Data Tables and Scenario Outlines

Level: Intermediate Run the same test with multiple data sets and pass structured data to steps.

1. Data Tables

A Data Table lets you pass structured tabular data directly inside a Gherkin step. It is attached to the step immediately below it. 

Feature: User Registration

  Scenario: Register a new user with full details
    Given the registration form is displayed
    When the user fills in the following details:
      | Field     | Value              |
      | First name | Jane              |
      | Last name  | Doe               |
      | Email      | jane@example.com  |
      | Password   | SecurePass1!      |
    Then the user account should be created

2. Reading a Data Table in Step Definitions

Cucumber passes the data table as a DataTable object. You can convert it to a map, a list of rows, or a list of objects depending on its shape.

A. Key-Value Table (rowsHash)

For a two-column table where column 1 is keys and column 2 is values:

import { When } from '@cucumber/cucumber';
import { DataTable } from '@cucumber/cucumber';
import { CustomWorld } from '../support/world';

When('the user fills in the following details:',
  async function (this: CustomWorld, dataTable: DataTable) {
    const data = dataTable.rowsHash(); // { 'First name': 'Jane', 'Email': '...', ... }

    await this.page.fill('[name="firstName"]', data['First name']);
    await this.page.fill('[name="lastName"]', data['Last name']);
    await this.page.fill('[name="email"]', data['Email']);
    await this.page.fill('[name="password"]', data['Password']);
  }
);

B. List of Maps (hashes) — table with headers

For multi-row tables where the first row is a header:

# Feature file
When the shopping cart contains the following items:
  | Product        | Quantity | Price  |
  | Wireless Mouse | 2        | $29.99 |
  | USB-C Hub      | 1        | $49.99 |
  | Keyboard       | 1        | $79.99 |
// Step definition
When('the shopping cart contains the following items:',
  async function (this: CustomWorld, dataTable: DataTable) {
    const rows = dataTable.hashes();
    // [{ Product: 'Wireless Mouse', Quantity: '2', Price: '$29.99' }, ...]

    for (const row of rows) {
      console.log(`${row.Product}: ${row.Quantity} x ${row.Price}`);
    }
  }
);

C. Raw Rows Array

Use .raw() to get the table as a plain 2D array of strings:

const rows = dataTable.raw();
// [['Product', 'Quantity', 'Price'], ['Wireless Mouse', '2', '$29.99'], ...]

3. Scenario Outline

A Scenario Outline is a template that runs the same scenario multiple times with different data sets. Use <placeholder> tokens in the scenario and supply the values in an Examples table. 

Feature: Login Validation

  Scenario Outline: Login with various credentials
    Given the user is on the login page
    When the user enters email "<email>" and password "<password>"
    Then the result should be "<expected>"

    Examples:
      | email              | password     | expected              |
      | admin@test.com     | correct123   | Dashboard             |
      | admin@test.com     | wrongpass    | Invalid credentials   |
      | notexist@test.com  | any123       | User not found        |
      | admin@test.com     |              | Password is required  |

Cucumber generates one scenario for each row in Examples. The run above produces 4 separate scenarios.

4. Step Definitions for Scenario Outlines

The step definitions for Scenario Outlines are identical to normal scenarios — you just use {string} parameters to receive the values: 

import { When, Then } from '@cucumber/cucumber';
import { expect } from '@playwright/test';
import { CustomWorld } from '../support/world';

When(
  'the user enters email {string} and password {string}',
  async function (this: CustomWorld, email: string, password: string) {
    await this.page.fill('[data-testid="email"]', email);
    await this.page.fill('[data-testid="password"]', password);
    await this.page.click('[data-testid="login-btn"]');
  }
);

Then(
  'the result should be {string}',
  async function (this: CustomWorld, expected: string) {
    if (expected === 'Dashboard') {
      await expect(this.page).toHaveURL(/dashboard/);
    } else {
      await expect(this.page.locator('.error-message')).toContainText(expected);
    }
  }
);

5. Tagging Examples Tables

You can tag individual Examples blocks to run specific data sets:

Scenario Outline: Login validation

  Examples: Happy path
    @smoke
    | email          | password   | expected  |
    | admin@test.com | correct123 | Dashboard |

  Examples: Error cases
    @regression
    | email          | password  | expected            |
    | admin@test.com | wrongpass | Invalid credentials |

6. When to Use Data Table vs Scenario Outline

Situation Use
Passing structured input to one step (form fields, items) Data Table
Running the whole scenario multiple times with different input/output Scenario Outline + Examples
Verifying a list of things exist on screen Data Table
Boundary value testing (min, max, invalid) Scenario Outline + Examples

7. Complete Worked Example: Product Search

# src/features/product-search.feature
Feature: Product Search

  Scenario Outline: Search returns correct results
    Given the user is on the product catalogue page
    When the user searches for "<query>"
    Then the result count should be "<count>" items
    And the first result name should contain "<first_result>"

    Examples:
      | query    | count | first_result |
      | laptop   | 12    | ProBook      |
      | mouse    | 8     | Wireless     |
      | monitor  | 5     | UltraWide    |
// src/steps/search.steps.ts
import { Given, When, Then } from '@cucumber/cucumber';
import { expect } from '@playwright/test';
import { CustomWorld } from '../support/world';

Given('the user is on the product catalogue page', async function (this: CustomWorld) {
  await this.page.goto(process.env.BASE_URL + '/products');
});

When('the user searches for {string}', async function (this: CustomWorld, query: string) {
  await this.page.fill('[data-testid="search"]', query);
  await this.page.press('[data-testid="search"]', 'Enter');
});

Then('the result count should be {string} items', async function (this: CustomWorld, count: string) {
  const items = this.page.locator('.product-card');
  await expect(items).toHaveCount(parseInt(count));
});

Then('the first result name should contain {string}', async function (this: CustomWorld, name: string) {
  const first = this.page.locator('.product-card').first().locator('.product-name');
  await expect(first).toContainText(name);
});

Practice Exercise

  1. Create a Scenario Outline for a password validation feature with at least four rows in the Examples table covering valid, too short, no uppercase, and no number passwords.
  2. Create a Scenario that uses a Data Table to fill a registration form with 5 fields. In the step definition, use dataTable.rowsHash() to fill each form field.
  3. Add two Examples blocks to a Scenario Outline — one tagged @smoke and one tagged @regression. Run only @smoke.
