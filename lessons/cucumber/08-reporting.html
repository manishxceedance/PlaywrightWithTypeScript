Lesson 8: Reporting — HTML, JSON, and Allure Reports

Level: Intermediate Generate professional test reports that communicate results to the whole team.

1. Built-in Cucumber Formatters

Cucumber ships with several formatters. Configure them in the format array inside cucumber.json: 

{
  "default": {
    "format": [
      "progress-bar",
      "html:reports/cucumber-report.html",
      "json:reports/cucumber-report.json",
      "junit:reports/cucumber-report.xml"
    ]
  }
}
Formatter Output Best for
progress-bar Terminal progress dots Local development runs
pretty Full step output in terminal Debugging individual failures
html:path Standalone HTML page Sharing results with stakeholders
json:path JSON file Input for Allure or custom dashboards
junit:path JUnit XML file Jenkins, GitHub Actions test results

2. Viewing the Built-in HTML Report

After a test run, open the report file in your browser:

# Windows
start reports/cucumber-report.html

# Mac
open reports/cucumber-report.html

# Linux
xdg-open reports/cucumber-report.html

The HTML report shows passed/failed/pending scenarios, step-level results, error messages, and any attachments (screenshots) you added with this.attach().

3. Multiple Profiles in cucumber.json

Define different profiles for local vs CI runs: 

{
  "default": {
    "requireModule": ["ts-node/register"],
    "require": ["src/steps/**/*.ts", "src/support/**/*.ts"],
    "paths": ["src/features/**/*.feature"],
    "format": ["progress-bar", "html:reports/report.html", "json:reports/report.json"]
  },
  "ci": {
    "requireModule": ["ts-node/register"],
    "require": ["src/steps/**/*.ts", "src/support/**/*.ts"],
    "paths": ["src/features/**/*.feature"],
    "format": ["progress-bar", "junit:reports/junit.xml", "json:reports/report.json"],
    "parallel": 4
  },
  "smoke": {
    "requireModule": ["ts-node/register"],
    "require": ["src/steps/**/*.ts", "src/support/**/*.ts"],
    "paths": ["src/features/**/*.feature"],
    "tags": "@smoke",
    "format": ["progress-bar", "html:reports/smoke-report.html"]
  }
}

Run a specific profile:

npx cucumber-js --profile ci
npx cucumber-js --profile smoke

4. Allure Report (Enhanced Visual Reports)

Allure generates rich, interactive HTML dashboards with charts, categories, and drill-down views. It is the most popular option for professional teams.

Install Allure packages:

npm install --save-dev allure-cucumberjs allure-commandline

Update cucumber.json:

{
  "default": {
    "format": [
      "allure-cucumberjs/reporter"
    ],
    "formatOptions": {
      "resultsDir": "allure-results"
    }
  }
}

Add npm scripts to package.json:

"scripts": {
  "test": "cucumber-js",
  "allure:generate": "allure generate allure-results --clean -o allure-report",
  "allure:open": "allure open allure-report",
  "allure:report": "npm run test && npm run allure:generate && npm run allure:open"
}

Install Allure CLI (one-time):

# Using npm
npm install -g allure-commandline

# Or using scoop on Windows
scoop install allure

Generate and view:

npm test
npx allure generate allure-results --clean -o allure-report
npx allure open allure-report

5. Attaching Screenshots to Reports

Any attachments added with this.attach() appear inline in both the built-in HTML report and in Allure. Always attach on failure at minimum: 

After(async function (this: CustomWorld, scenario) {
  if (scenario.result?.status === Status.FAILED) {
    // Full-page screenshot
    const screenshot = await this.page.screenshot({ fullPage: true });
    this.attach(screenshot, 'image/png');

    // Current URL for debugging
    this.attach(`Failed URL: ${this.page.url()}`, 'text/plain');

    // Console errors (if any)
    const errors = this.consoleLogs.filter(l => l.type() === 'error');
    if (errors.length) {
      this.attach(errors.map(e => e.text()).join('\n'), 'text/plain');
    }
  }
  await this.context.close();
});

6. Adding Custom Allure Labels

You can add metadata to Allure reports using the allure instance in hooks or steps: 

import { allure } from 'allure-cucumberjs';

Before(async function (this: CustomWorld, scenario) {
  allure.label('feature', scenario.pickle.uri.split('/').pop() ?? '');
  allure.label('severity', scenario.pickle.tags.includes('@critical') ? 'critical' : 'normal');
});

7. Interpreting Report Results

A Cucumber HTML report shows five possible step states:

  • Passed — step ran and assertions succeeded.
  • Failed — an assertion failed or an error was thrown.
  • Pending — step returned 'pending'.
  • Undefined — no step definition matched the step text.
  • Skipped — step was not executed because a previous step failed.

Undefined steps are different from failed steps. An undefined step means the step definition is missing or the pattern does not match — fix the pattern first before debugging the automation logic.

Practice Exercise

  1. Configure your cucumber.json to output both HTML and JSON reports.
  2. Run your test suite, open the HTML report, and identify one passed and one failed (or pending) scenario.
  3. Add a screenshot-on-failure attachment in your After hook. Deliberately break one test and confirm the screenshot appears in the report.
  4. Install Allure, generate a report, and compare it to the built-in HTML report.
  5. Create a smoke profile in cucumber.json and run it with --profile smoke.
