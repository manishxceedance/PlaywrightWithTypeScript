Lesson 4: Hooks and Tags — Setup, Teardown, and Filtering

Level: Intermediate Control what runs before and after scenarios, and filter tests with tags.

1. What Are Hooks?

Hooks are functions that run automatically at specific points in the test lifecycle. They are defined in TypeScript files (usually inside src/support/) and are picked up by Cucumber through the require paths in cucumber.json.

The six commonly used hook types in Cucumber.js are:

  • BeforeAll — runs once before the entire test suite.
  • AfterAll — runs once after the entire test suite.
  • Before — runs before each scenario.
  • After — runs after each scenario.
  • BeforeStep — runs before each step in a scenario.
  • AfterStep — runs after each step in a scenario.

2. All Hook Types with Proper Definitions

Hook Runs When Best Use
BeforeAll Exactly once before any scenario starts. Suite-level setup: launch browser, seed shared config, initialize global resources.
AfterAll Exactly once after all scenarios finish. Suite-level teardown: close browser, release shared resources, final reporting cleanup.
Before Before each scenario. Scenario isolation: create fresh context/page, set viewport, clear scenario state.
After After each scenario. Cleanup and diagnostics: screenshots on failure, attach logs, close scenario context.
BeforeStep Before each step inside a scenario. Step-level observability: lightweight logging, tracing current step text.
AfterStep After each step inside a scenario. Step-level diagnostics: attach per-step screenshots/logs only when a step fails.

3. The Essential Browser Lifecycle Hook

The most important hook in a Playwright + Cucumber project launches and closes the browser. This goes in src/support/hooks.ts: 

// src/support/hooks.ts
import { Before, After, BeforeAll, AfterAll, Status } from '@cucumber/cucumber';
import { chromium, Browser } from '@playwright/test';
import { CustomWorld } from './world';

let sharedBrowser: Browser;

BeforeAll(async function () {
  sharedBrowser = await chromium.launch({ headless: true });
});

Before(async function (this: CustomWorld) {
  this.context = await sharedBrowser.newContext();
  this.page = await this.context.newPage();
});

After(async function (this: CustomWorld, scenario) {
  // Take screenshot on failure
  if (scenario.result?.status === Status.FAILED) {
    const screenshotName = scenario.pickle.name.replace(/\s+/g, '-');
    const screenshot = await this.page.screenshot({ fullPage: true });
    this.attach(screenshot, 'image/png');
  }
  await this.context.close();
});

AfterAll(async function () {
  await sharedBrowser.close();
});

4. The this.attach() Method

Inside hooks and step definitions, this.attach() embeds data into the Cucumber report. You can attach screenshots, text, or JSON: 

// Attach a screenshot (Buffer)
const screenshot = await this.page.screenshot();
this.attach(screenshot, 'image/png');

// Attach a text message
this.attach('User was redirected to dashboard', 'text/plain');

// Attach JSON data
this.attach(JSON.stringify({ userId: 42 }), 'application/json');

5. Tagged Hooks

You can run a hook only when a scenario has a specific tag. This is useful for scenarios that need a logged-in user without making all tests go through login. 

import { Before } from '@cucumber/cucumber';
import { CustomWorld } from './world';

// Only runs for scenarios tagged @authenticated
Before({ tags: '@authenticated' }, async function (this: CustomWorld) {
  await this.page.goto('https://manishxceedance.github.io/PlaywrightWithTypeScript/lessons/practice-app/hrms/login.html');
  await this.page.fill('#username', 'admin');
  await this.page.fill('#password', 'admin');
  await this.page.click('button[type="submit"]');
  await this.page.waitForURL(/hrms/);
});

// Only runs for scenarios tagged @api
Before({ tags: '@api' }, async function (this: CustomWorld) {
  // Set up API request context
});

6. Hook Order and Priority

When multiple Before hooks match a scenario, they run in the order they are registered. You can control the order with order: 

// Runs first (lower number = earlier)
Before({ order: 1 }, async function (this: CustomWorld) {
  this.context = await sharedBrowser.newContext();
  this.page = await this.context.newPage();
});

// Runs second
Before({ order: 2, tags: '@authenticated' }, async function (this: CustomWorld) {
  await loginAsAdmin(this.page);
});

7. Step-Level Hooks (BeforeStep and AfterStep)

Step hooks are powerful for debugging because they execute around every step. Keep them lightweight to avoid slowing test execution. 

import { BeforeStep, AfterStep, Status } from '@cucumber/cucumber';
import { CustomWorld } from './world';

BeforeStep(async function (this: CustomWorld, step) {
  // Good: lightweight trace of what is about to run
  this.attach(`Starting step: ${step.pickleStep.text}`, 'text/plain');
});

AfterStep(async function (this: CustomWorld, step) {
  // Good: capture evidence only when a step fails
  if (step.result?.status === Status.FAILED) {
    const shot = await this.page.screenshot({ fullPage: true });
    this.attach(shot, 'image/png');
    this.attach(`Failed step: ${step.pickleStep.text}`, 'text/plain');
  }
});

8. Tags in Depth

Tags can be applied at Feature level, Scenario level, or Scenario Outline level:

@smoke
Feature: Login

  @regression @critical
  Scenario: Valid login
    ...

  @wip
  Scenario: SSO login
    ...

A scenario inherits all tags from its Feature. So the "Valid login" scenario above has the tags @smoke, @regression, and @critical.

9. Running Tests by Tag

# Run all smoke tests
npx cucumber-js --tags "@smoke"

# Run smoke AND regression tests
npx cucumber-js --tags "@smoke and @regression"

# Run smoke OR regression tests
npx cucumber-js --tags "@smoke or @regression"

# Run all tests except WIP
npx cucumber-js --tags "not @wip"

# Run smoke but not critical
npx cucumber-js --tags "@smoke and not @critical"

In PowerShell, always wrap the tag expression in quotes to prevent PowerShell from interpreting @ as an array operator: 

npx cucumber-js --tags "@smoke"

10. Scenario Status in the After Hook

The After hook receives a scenario object with the result. Use it to clean up differently based on pass or fail: 

import { After, Status } from '@cucumber/cucumber';
import { CustomWorld } from './world';

After(async function (this: CustomWorld, scenario) {
  const status = scenario.result?.status;

  if (status === Status.FAILED) {
    console.log(`FAILED: ${scenario.pickle.name}`);
    const screenshot = await this.page.screenshot({ fullPage: true });
    this.attach(screenshot, 'image/png');
  }

  if (status === Status.PASSED) {
    console.log(`PASSED: ${scenario.pickle.name}`);
  }

  await this.context.close();
});

11. Complete hooks.ts Template (All Hooks)

// src/support/hooks.ts
      import {
        Before,
        After,
        BeforeAll,
        AfterAll,
        BeforeStep,
        AfterStep,
        Status
      } from '@cucumber/cucumber';
import { chromium, Browser } from '@playwright/test';
import { CustomWorld } from './world';

let browser: Browser;

BeforeAll(async function () {
  browser = await chromium.launch({
    headless: process.env.HEADED !== 'true'
  });
});

Before(async function (this: CustomWorld) {
  this.context = await browser.newContext({
    viewport: { width: 1280, height: 720 }
  });
  this.page = await this.context.newPage();
});

Before({ tags: '@authenticated' }, async function (this: CustomWorld) {
  await this.page.goto(process.env.BASE_URL + '/hrms/login.html');
  await this.page.fill('[data-testid="login-username"]', process.env.TEST_USER!);
  await this.page.fill('[data-testid="login-password"]', process.env.TEST_PASS!);
  await this.page.click('[data-testid="login-submit"]');
  await this.page.waitForURL(/hrms/);
});

BeforeStep(async function (this: CustomWorld, step) {
  this.attach(`STEP START: ${step.pickleStep.text}`, 'text/plain');
});

AfterStep(async function (this: CustomWorld, step) {
  if (step.result?.status === Status.FAILED) {
    const shot = await this.page.screenshot({ fullPage: true });
    this.attach(shot, 'image/png');
    this.attach(`STEP FAILED: ${step.pickleStep.text}`, 'text/plain');
  }
});

After(async function (this: CustomWorld, scenario) {
  if (scenario.result?.status === Status.FAILED) {
    const screenshot = await this.page.screenshot({ fullPage: true });
    this.attach(screenshot, 'image/png');
  }
  await this.context.close();
});

AfterAll(async function () {
  await browser.close();
});

12. Hook Exercise Using the Sample Apps

This exercise intentionally uses multiple practice pages so you can see where each hook is useful. 

@smoke @basic
Scenario: Fill basic controls form
  Given the user opens the basic controls page
  When the user fills text, email, number and select fields
  Then the form actions should be successful

@regression @authenticated
Scenario: Search employees after login
  Given the user opens the HRMS employees page
  When the user searches for "Emma"
  Then at least one employee row should be visible

@regression @dialogs
Scenario: Handle alert and confirm
  Given the user opens the dialogs page
  When the user triggers an alert and accepts it
  And the user triggers a confirm and dismisses it
  Then dialog results should be displayed

@regression @frame @shadow
Scenario: Use iframe and shadow controls
  Given the user opens the register page step five
  When the user selects a date in the iframe calendar
  And the user enters tags in the shadow component
  Then profile preview should update

Required hook implementation tasks:

  1. BeforeAll: launch one browser for the suite.
  2. Before: create a fresh context and page per scenario.
  3. Before tagged @authenticated: login to HRMS with admin/admin.
  4. BeforeStep: attach the current step text as plain text.
  5. AfterStep: on failed step, attach screenshot and failed step text.
  6. After: always close context and attach scenario-level screenshot on failure.
  7. AfterAll: close shared browser.

Expected learning outcome: You can explain exactly why each hook belongs at suite, scenario, or step scope, and when to use tagged hooks instead of hard-coding setup in step definitions.

Practice Exercise

  1. Create src/support/hooks.ts with all six hooks: BeforeAll, AfterAll, Before, After, BeforeStep, AfterStep.
  2. Implement the @authenticated tagged hook that logs into hrms/login.html using admin/admin.
  3. Run only @dialogs scenarios and verify step-level logs from BeforeStep are attached.
  4. Force one failing step in the iframe or shadow scenario and verify AfterStep attaches screenshot evidence.
  5. Run full suite and confirm browser launches once (BeforeAll) but context/page are recreated per scenario (Before).
