Lesson 0: JavaScript Basics Before TypeScript

Level: Absolute Beginner No programming background required. Learn these basics first.

TypeScript is built on top of JavaScript. If you are new to coding, this lesson gives you the exact concepts you must understand before writing TypeScript.

0. Before Coding: How to Run a File and Read Syntax

New learners often struggle not because of logic, but because they do not know how to execute code. Start here.

  • Create a file named basics.js.
  • Write one line: console.log("Hello, JavaScript").
  • Open terminal in the same folder and run node basics.js.
  • If you see output, your environment is working.

How to read a statement: console.log("Hello") means call the log function from console and print text.

1. What is a Variable?

A variable is a named box that stores data. You create a variable, put a value in it, and reuse it later. 

let testerName = "Riya";
let totalCases = 30;
console.log(testerName, totalCases);

2. Difference Between var, let, and const

This is one of the most important beginner topics.

A. var (old style, avoid in modern code)

var is function-scoped and can be re-declared in the same scope, which can create bugs. 

var environment = "qa";
var environment = "staging"; // allowed, but risky
console.log(environment);

B. let (preferred for values that change)

let is block-scoped and can be reassigned, but cannot be re-declared in the same block. 

let passed = 12;
passed = 13; // allowed
// let passed = 14; // not allowed in the same block

C. const (preferred by default)

const is block-scoped and cannot be reassigned. Use const unless you know the variable itself will change. 

const projectName = "Billing Portal";
// projectName = "Payments Portal"; // error

Important beginner note: with const object variables, object properties can still change. 

const result = { passed: 8 };
result.passed = 9; // allowed
// result = { passed: 10 }; // not allowed

Quick rule for beginners

  • Use const by default.
  • Use let if value must change.
  • Avoid var in modern TypeScript/JavaScript projects.

3. What is Scope? (Why let/const are safer)

Scope means where a variable is accessible. Block scope is code inside { }. 

if (true) {
  let insideBlock = "visible only here";
  const anotherOne = "also only here";
}

// console.log(insideBlock); // error

4. Basic Data Types in JavaScript

JavaScript has 7 primitive types and 1 non-primitive base type. Arrays and functions are also objects.

Type Category Description
string Primitive Textual data
number Primitive Integers and decimals
boolean Primitive true or false
bigint Primitive Large integers
null Primitive Empty or absent value
undefined Primitive Declared but not assigned
symbol Primitive Unique identifier
object Non-Primitive Complex collections 
const status = "passed";                  // string
const responseCode = 200;                 // number
const isCritical = true;                  // boolean
const hugeCount = 9007199254740993n;      // bigint
const uniqueToken = Symbol("test-run");  // symbol
const notLoaded = null;                   // null
let owner;                                // undefined

const tags = ["smoke", "api"];          // object (array)
const testCase = { id: "TC-10", title: "Login" }; // object

5. What is get? (Getter in Objects/Classes)

get creates a getter. A getter lets you read a computed value like a property, without calling it like a function. 

const report = {
  passed: 45,
  failed: 5,
  get total() {
    return this.passed + this.failed;
  }
};

console.log(report.total); // 50 (notice: no parentheses)

Why this is useful: you can keep logic in one place and expose a clean read-only style value.

Compare the two styles below. Both work, but getter style is cleaner at usage point. 

const reportA = {
  passed: 10,
  failed: 2,
  total() {
    return this.passed + this.failed;
  }
};

const reportB = {
  passed: 10,
  failed: 2,
  get total() {
    return this.passed + this.failed;
  }
};

console.log(reportA.total()); // method call
console.log(reportB.total);   // getter property

Related concept: set lets you control how a value is updated. 

const scoreCard = {
  _score: 0,
  get score() {
    return this._score;
  },
  set score(value) {
    if (value < 0) {
      this._score = 0;
      return;
    }
    this._score = value;
  }
};

scoreCard.score = 15;
console.log(scoreCard.score); // 15

6. Simple If, Loop, and Function (Minimum Required Basics)

const defects = 2;

if (defects > 0) {
  console.log("Retest required");
}

for (let i = 1; i <= 3; i += 1) {
  console.log(`Running test cycle ${i}`);
}

function add(a, b) {
  return a + b;
}

console.log(add(2, 3));

7. How this connects to TypeScript

In TypeScript, the same JavaScript syntax is used, but you also define types to prevent mistakes. 

const tester: string = "Riya";
let cycleCount: number = 1;

cycleCount = cycleCount + 1;
