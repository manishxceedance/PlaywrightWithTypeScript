Lesson 1: Start from Scratch - Prerequisites and Project Setup

Level: Beginner Zero experience assumed. Follow these steps in sequence.

This lesson is for absolute beginners. By the end, you will have a running TypeScript project and understand each setup step.

1. Why TypeScript for Manual Testers?

TypeScript helps you catch defects early while writing scripts, test utilities, and API data checks. Instead of finding mistakes only at runtime, TypeScript reports many issues while coding.

2. Prerequisites (Install These First)

A. Node.js and npm

TypeScript compiler is installed through npm, which comes with Node.js. 

node -v
npm -v

If both commands return version numbers, your environment is ready.

B. Visual Studio Code

Install VS Code and recommended extensions:

  • ESLint
  • Prettier - Code formatter
  • TypeScript and JavaScript Language Features (usually built-in)

C. Basic terminal knowledge

You should know how to open a terminal and run commands in your project folder.

3. Create Your First TypeScript Project

Create a folder and move into it:

mkdir ts-from-scratch
cd ts-from-scratch

Initialize npm package:

npm init -y

Install TypeScript as a dev dependency:

npm install -D typescript

Create TypeScript configuration:

npx tsc --init

After creating tsconfig.json, set these beginner-friendly options so compile output always goes to dist. 

{
  "compilerOptions": {
    "rootDir": "src",
    "outDir": "dist",
    "strict": true
  },
  "include": ["src"]
}

4. Recommended Beginner Folder Structure

ts-from-scratch/
  src/
    index.ts
  dist/
  package.json
  tsconfig.json

Create the folder and file with commands:

mkdir src
  New-Item src/index.ts -ItemType File

Create source file and add sample code:

// src/index.ts
const testerName: string = "Asha";
const executedCases: number = 24;

console.log(`${testerName} executed ${executedCases} test cases today.`);

5. Compile and Run

Compile TypeScript into JavaScript:

npx tsc

Run generated JavaScript:

node dist/index.js

If your output path is different, check your tsconfig.json values for rootDir and outDir.

First-run troubleshooting (important for beginners)

  • npx tsc says file not found: verify src/index.ts exists.
  • node dist/index.js fails: compile again and confirm outDir is dist.
  • PowerShell says command not found: close and reopen terminal after installing Node.js.

6. Add Helpful npm Scripts

Update your package scripts:

{
  "scripts": {
    "build": "tsc",
    "start": "node dist/index.js",
    "dev": "tsc --watch"
  }
}

Now your daily commands are easier:

npm run build
npm run start

7. Common Setup Mistakes

  • Node.js not installed, so npm commands fail.
  • Running commands outside project folder.
  • Wrong file path in run command (for example, dist/index.js not found).
  • Not creating src/index.ts before compile.
