Lesson 2: Core Types, Unions, and Narrowing

Level: Beginner Build your typing foundation before moving to functions and OOP.

Strong typing is the first big TypeScript advantage. In this lesson, you’ll learn how to define and safely use all the core types you’ll see in real projects.

1. Basic Types

TypeScript lets you specify exactly what kind of value a variable can hold. The colon means "type annotation". Example: envName: string means variable name is envName and its type must be string.

// Basic types
const envName: string = "staging";
const retryCount: number = 3;
const isRegressionRun: boolean = true;

// Array of strings
const tags: string[] = ["smoke", "api", "critical"];

2. Tuples

A tuple is a special array where you know exactly how many items there are, and what type each item should be. Think of it as a fixed-size, ordered list. Use tuples for things like coordinates, RGB colors, or fixed-format data rows.

// A tuple for a test result: [test case id, status, duration]
const result: [string, string, number] = ["TC-101", "pass", 2.5];
// You must provide exactly three items, in the right order and type.
// result[0] is a string (id), result[1] is a string (status), result[2] is a number (duration)

// Example: representing a coordinate
const point: [number, number] = [10, 20];

3. Enums

An enum (short for "enumeration") lets you define a set of named constants. Use enums when you want a variable to have one of a few fixed values, and you want those values to have readable names.

// Define an enum for test status
enum Status {
  Passed = "passed",
  Failed = "failed",
  Blocked = "blocked"
}
const runStatus: Status = Status.Passed;
console.log(runStatus); // "passed"

4. Union Types

A union type lets a variable accept more than one type. This is useful when a value could be a string or a number, for example.

let testIdentifier: string | number;
testIdentifier = "TC-101";
testIdentifier = 101;

5. Type Narrowing

Before using a value, narrow it so TypeScript knows which branch is safe. This means checking the type before using it in a way that only works for one type.

function printTestIdentifier(id: string | number): void {
  if (typeof id === "string") {
    console.log(id.toUpperCase());
  } else {
    console.log(id.toFixed(0));
  }
}

6. Literal and Optional Values

Literal types let you specify exact values a variable can have. Optional types let you say a value might be missing (with ?).

// Literal type
type RunStatus = "passed" | "failed" | "blocked";
let status: RunStatus = "passed";
// status = "skipped"; // Error

// Optional property in an object
type TestCase = {
  id: string;
  description?: string; // optional
};

7. Type Assertions

Type assertion tells TypeScript, "Trust me, I know the type of this value." Use it when you know more than the compiler, but use it carefully!

// Example: reading from a value that could be string or number
let value: unknown = "123";
// Tell TypeScript to treat value as a string
let strValue = value as string;
console.log(strValue.length); // 3

// You can also use the angle-bracket syntax (not in JSX files):
let numValue = (value as unknown as number);
  • Type assertions do not change the runtime value, only how TypeScript checks it.
  • Use assertions only when you are sure of the type, or you might get runtime errors.

8. Type Inference

Type inference means TypeScript can figure out the type for you, even if you don’t write it. This makes your code shorter and still safe.

// TypeScript knows this is a string
let message = "Hello";
// message: string

// TypeScript knows this is a number
let count = 42;
// count: number

9. Never and Unknown Types

never is a type for values that never happen (like a function that always throws or never returns). unknown is a safe type for values you don’t know yet—you must check before using it.

// never: a function that always throws
function fail(message: string): never {
  throw new Error(message);
}

// unknown: must check type before using
let input: unknown = "hello";
if (typeof input === "string") {
  console.log(input.toUpperCase());
}
  • Use never for impossible cases or functions that never finish.
  • Use unknown for values from outside (like user input or APIs) when you want safety.

10. Type Guards

Type guards are ways to check the type of a value at runtime, so TypeScript knows what you’re working with. This keeps your code safe and avoids errors.

// Using typeof
function printId(id: string | number) {
  if (typeof id === "string") {
    console.log("String id:", id.toUpperCase());
  } else {
    console.log("Number id:", id.toFixed(2));
  }
}

// Using 'in' for objects
type Cat = { meow: () => void };
type Dog = { bark: () => void };
function speak(animal: Cat | Dog) {
  if ("meow" in animal) {
    animal.meow();
  } else {
    animal.bark();
  }
}
  • Use typeof for primitives, in for object properties, and instanceof for classes.

A tuple is a special array where you know exactly how many items there are, and what type each item should be. Think of it as a fixed-size, ordered list. 

// A tuple for a test result: [test case id, status, duration]

// Example: representing a coordinate

// Why use tuples? When you want to group a small, fixed set of values together, and their order matters.
  • Tuples are stricter than arrays: [string, number] is not the same as [number, string].
  • Use tuples for things like coordinates, RGB colors, or fixed-format data rows.

3. Enums

An enum (short for "enumeration") lets you define a set of named constants. Use enums when you want a variable to have one of a few fixed values, and you want those values to have readable names.

}

const runStatus: Status = Status.Passed;
console.log(runStatus); // "passed"
  • Enums make your code more readable and less error-prone than using plain strings.
  • Use enums for things like status codes, user roles, or fixed categories.

5. Practical Tester Example

function summarize(result: TestResult): string {
  const defectText = result.defectId ? `, defect: ${result.defectId}` : "";
  return `${result.caseId} -> ${result.status}${defectText}`;
}
